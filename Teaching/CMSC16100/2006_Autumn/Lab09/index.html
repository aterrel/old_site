Lab 9 - Eliza: Natural Language Emulation

Introduction

This week you will be implementing the famous Eliza program. This lab is going to be open ended, meaning I'm not going to tell you what to implement so much as give you some guidelines for getting started and let you make the program. There are several implementations on the web and even one in Emacs to give you some ideas on what to do, I'll show you at least one in lab and you can use that for some ideas.

Eliza description

The Eliza program takes a string, returns a string, and waits for another string. The idea is to create an experience similar to that of a psychotherapist, by parsing the string and turning it into a question of some sort. For example, if I enter "I hate Scheme!", Eliza might reply "Why do you hate Scheme?". This was done by turning 'I' to 'you' and then appending the action phrase 'hate Scheme' onto 'Why do'+'you'. There are lots of ways to do different things, such as ask "Do you hate Scheme because of issues in your life?" I will let you have fun with coming up with different ways to match the string and respond. I should be able to say "goodbye" in order to stop the program.

Notes on getting things to work

In order to get things running you will need a function that will read input from the user, reply and then ask for more input. I recommend that you don't take just strings but rather use lists of symbols. For example "I hate Scheme" should be inputted at (I hate Scheme).

You will need to use the (read) function in order to get input from the user. Make sure you can read how what is inputted ... ie play around with functions to make sure you are reading input correctly.

You should then give your string to a function that will try to parse it and respond. If that function is unable to parse the string you should return false and give it to another function that might succeed and so on. Finally you can resort to either asking for more details or just give some random message.

Documentation

I expect your programs to be documented so that I know exactly what you have implemented. If there is no documentation I cannot judge if your code is doing what you intended and thus am unable to grade it ... which makes a very poor grade for you.

Turning Everything in

So for example I would save my file as aterrel-Lab09.scm. Also do yourself a favour and save your file somewhere you can look back at it (email yourself or store it on your own media).
